# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Lilith 是 Mirror Media 系列媒體品牌的 CMS monorepo，基於 **KeystoneJS 6** 與 **yarn workspaces**。

## Installation

```bash
yarn install
```

若安裝失敗（`postinstall` 錯誤），依序在以下目錄執行 `yarn build`，再重跑 root `yarn install`：
1. `packages/draft-renderer`
2. `packages/draft-editor`
3. `packages/core`

## Common Commands

### Library Packages (`core`, `draft-editor`, `draft-renderer`)
```bash
cd packages/<pkg>
make build    # babel transpile + tsc type definitions
make clean    # remove lib/ and @types/
```

### CMS Application Packages
```bash
cd packages/<cms-pkg>
yarn dev              # start dev server (http://localhost:3000)
yarn build            # production build
yarn start            # run production build
yarn db-migrate       # apply pending prisma migrations
```

### Database (local dev)
```bash
docker run -p 5433:5432 --name lilith-<pkg> \
  -e POSTGRES_PASSWORD=passwd \
  -e POSTGRES_USER=account \
  -e POSTGRES_DB=lilith-<pkg> \
  -d postgres
DATABASE_URL=postgres://account:passwd@localhost:5433/lilith-<pkg> yarn dev
```

### Running a specific package from root
```bash
yarn workspace @mirrormedia/lilith-mirrormedia dev
```

## Architecture

### Package Roles

| Package | Type | Purpose |
|---|---|---|
| `core` | Library | Shared utilities, custom fields, access control — consumed by all CMS packages |
| `draft-renderer` | Library | Draft.js rich text renderer (React components) |
| `draft-editor` | Library | Draft.js rich text editor (Keystone custom field UI) |
| `mirrormedia` | CMS App | Mirror Media 週刊 CMS |
| `mirrortv` | CMS App | Mirror TV CMS |
| `mirrordaily` | CMS App | Mirror Daily CMS |
| `readr` | CMS App | Readr CMS |
| `mesh` | CMS App | Mesh CMS |
| `openrelationship` | CMS App | Open Relationship CMS |
| `advertise` | CMS App | Advertising management CMS |
| `editools` | CMS App | Editorial tools CMS |

### CMS App Structure (per package)

```
packages/<cms-pkg>/
├── keystone.ts          # KeystoneJS config: auth, session, GraphQL, Apollo cache
├── environment-variables.ts  # All env vars with defaults
├── lists/               # Data models (one file per Keystone list)
│   ├── index.ts         # Re-exports all lists
│   └── *.ts             # Individual list definitions
├── migrations/          # Prisma migration SQL files
├── admin/               # Admin UI customizations
├── express-mini-apps/   # Express sub-apps (preview server, image auth, etc.)
└── schema.prisma        # Auto-generated by Keystone
```

See **SPEC.md** for detailed architecture, middleware ordering, and feature specs.

### Core Package Utilities (`packages/core/src/`)

- **`utils/accessControl.ts`** — Role-based access control with multiple strategies. Roles: `admin`, `moderator`, `editor`, `contributor`. Strategies controlled by `ACCESS_CONTROL_STRATEGY` env var: `cms` (default), `gql`, `preview`, `restricted`.
- **`utils/trackingHandler.ts`** — `addTrackingFields()`: adds `createdAt`/`updatedAt`/`createdBy`/`updatedBy` fields and auto-populates via hooks.
- **`utils/invalidate-cache-after-operation.ts`** — CDN cache invalidation hook after mutations.
- **`utils/manual-order-relationship.ts`** — `addManualOrderRelationshipFields()`: manual drag-and-drop ordering for relationship fields.
- **`custom-fields/rich-text-editor/`** — Keystone custom field wrapping draft-editor.
- **`custom-fields/select-with-color/`** — Keystone custom select field with color indicators.

### Access Control Strategies

| Strategy | Behavior |
|---|---|
| `cms` | Standard role-based access for CMS editors |
| `gql` | Public read (for frontend APIs); returns only published items |
| `preview` | All items readable (for preview servers) |
| `restricted` | Blocks User list queries/updates/deletes by default; fine-tuned via additional env vars |

### Adding a New List to a CMS Package

1. Create `packages/<pkg>/lists/<ModelName>.ts` — define with `list()` from `@keystone-6/core`
2. Add access control using `utils.accessControl.allowRoles(admin, moderator, editor)`
3. Apply `addTrackingFields()` from `@mirrormedia/lilith-core` if tracking is needed
4. Export the list in `packages/<pkg>/lists/index.ts`
5. Run `yarn dev` — Keystone auto-generates a migration prompt; accept to create migration SQL in `migrations/`

### Build & Deploy

- CI/CD via **Google Cloud Build** (`cloudbuild.yaml` at root)
- Each CMS package has its own `Dockerfile`
- Builds copy root `yarn.lock` and `patches/` into the package before building Docker image
- Deployed to **Google Cloud Run**

## Key Environment Variables (CMS packages)

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string |
| `ACCESS_CONTROL_STRATEGY` | `cms` / `gql` / `preview` / `restricted` |
| `IS_UI_DISABLED` | `true` to run as GraphQL-only API server |
| `GCS_BUCKET` | Google Cloud Storage bucket for media |
| `REDIS_SERVER` | Redis URL for Apollo response cache |
| `IS_CACHE_ENABLED` | Enable Apollo cache with Redis |
| `PREVIEW_SERVER_ORIGIN` | Preview server URL |
| `INVALID_CDN_CACHE_SERVER_URL` | CDN cache invalidation endpoint |
